ReAct NLU v2 execution flow

Domain prefetch → intent + slot → per-slot rewrite → validation

Phase 0 — Domain knowledge prefetch
T
Thought Analyze query & plan domain retrieval ›

Receive user query. Before any intent/slot work, determine what domain knowledge is needed to ground the analysis.

// Thought: Query: "{input}" I need to fetch reading-domain knowledge before classifying intent or extracting slots.
A
Action Call query domain search + vertical KB retrieval ›

Two parallel tool calls to build domain context:

Action 1: query_domain_search("{input}") → books, authors, categories related to query Action 2: vertical_kb_retrieval(keywords) → reading-domain background entities
O
Observation Domain context collected ›

Merge tool results into a Domain Context object — this anchors all downstream reasoning to the reading vertical.

Domain Context = { related_books: [...], related_authors: [...], categories: [...], entities: [...] }
Phase 1 — Intent classification + slot extraction
T
Thought Fuse internal knowledge + domain context ›

Classify intent (author / book / category / recommendation) and extract slots, using domain context to prefer standard entity names. Each slot gets a domain status tag.

Confirmed in domain Needs verification Not in domain → rewrite
Intent: book_intent Slot 1: "value_a" ✓ confirmed Slot 2: "value_b" ! needs verification Slot 3: "value_c" × not in domain
Phase 2 — Per-slot tool verification + domain rewrite
A
Action Verify each flagged slot via tools ›

For every slot marked as ⚠ or ❌, call the slot search tool individually to check domain membership.

--- Slot B --- Action: slot_search("value_b") Result: belongs to film domain, not reading --- Slot C --- Action: slot_search("value_c") Result: music artist, not reading
R
Rewrite Map non-domain slots to reading entities ›

For slots outside the reading domain, find the corresponding reading-domain entity. If no mapping exists, mark as unmapped and preserve original.

Slot B: "The Shawshank Redemption" (film) → rewrite: "Rita Hayworth and Shawshank Redemption" (novella) Slot C: "Jay Chou" (musician) → rewrite: unmapped (no reading-domain match)
O
Observation Rewritten slots collected ›
Slot 1: "value_a" ✓ confirmed Slot 2: "rewritten_b" ↻ rewritten (was: "value_b") Slot 3: "value_c" × unmapped
Phase 3 — Validation
V
Retry if invalid → back to Step 1
Validation Five-point result check ›
1. Intent check → pass semantically consistent 2. Slot completeness → pass no missing slots 3. Slot consistency → pass no conflicts 4. Rewrite accuracy → pass mapping is correct 5. Ambiguity check → pass single interpretation

If any check fails, loop back to the corresponding step and re-execute the downstream flow.

Output — Final answer
F
Final Structured JSON output ›
{ "intent": "book_intent", "confidence": 0.95, "slots": { "book_name": { "value": "Rita Hayworth and Shawshank Redemption", "original_value": "The Shawshank Redemption", "type": "book_title", "domain_status": "rewritten", "rewrite_reason": "Film title mapped to original novella" } }, "domain_context_used": true, "validation_passed": true }
