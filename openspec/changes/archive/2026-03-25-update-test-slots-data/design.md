## Context

当前项目已经建立了完整的数据插入工具链，包括ES数据插入助手和测试数据文件。现有的测试数据使用通用字段名称，虽然功能上可以正常工作，但缺乏领域特异性，无法充分测试真实业务场景下的数据表现。

## Goals / Non-Goals

**Goals:**
- 创建具有阅读领域特色的测试数据
- 包含30个真实的作者和书名组合
- 保持数据格式和结构的兼容性
- 覆盖各种边界情况和数据类型
- 提供更真实的测试场景

**Non-Goals:**
- 不修改数据插入工具的核心功能
- 不改变现有的文件格式和结构
- 不引入新的依赖或技术栈
- 不影响现有的测试用例结构

## Decisions

### 1. 数据内容选择
- **选择标准**: 使用知名作家和经典作品，确保数据的真实性和代表性
- **多样性**: 包含中外作家、不同年代、不同文学流派
- **格式保持**: 保持原有的slotName, slotReName, slotType字段结构

### 2. 数据生成策略
- **手动精选**: 选择30个有代表性的作者-作品组合
- **格式兼容**: 确保CSV和TSV格式的完全兼容
- **边界覆盖**: 包含长书名、特殊字符、多作者等情况

### 3. 实施方法
- **直接替换**: 直接更新现有的test_slots.csv和test_slots.tsv文件
- **版本控制**: 保留原有测试数据备份，便于需要时回退
- **文档更新**: 更新相关注释说明新的测试数据内容

## Risks / Trade-offs

**Risks:**
- 数据真实性风险 → 通过选择知名作品确保准确性
- 格式兼容性风险 → 严格保持原有字段结构和分隔符
- 测试用例依赖风险 → 确保新数据不影响现有测试逻辑

**Trade-offs:**
- 手动精选 vs 自动生成 → 选择手动精选确保质量
- 全面性 vs 实用性 → 选择30个有代表性的组合
- 领域特异性 vs 通用性 → 优先领域特异性以支持阅读相关功能开发
