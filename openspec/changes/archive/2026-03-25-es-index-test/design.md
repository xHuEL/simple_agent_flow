## Context

当前项目需要验证Elasticsearch服务的可用性和索引创建功能。基于之前创建的Docker ES部署指南，现在需要添加自动化测试来确保索引结构正确创建。

## Goals / Non-Goals

**Goals:**
- 创建测试脚本验证ES索引创建功能
- 测试特定索引结构（text slotName, keyword slotReName, slotType字段）
- 集成到现有pytest测试框架中
- 提供基本的索引操作验证

**Non-Goals:**
- 不涉及复杂的ES查询测试
- 不包含性能测试
- 不涉及集群配置测试

## Decisions

1. **测试框架选择**: 使用现有pytest框架，保持测试一致性
2. **ES客户端选择**: 使用elasticsearch-py官方客户端库
3. **测试数据管理**: 使用临时测试索引，测试完成后自动清理
4. **错误处理**: 包含详细的错误信息和重试机制
5. **配置管理**: 通过环境变量配置ES连接信息

## Risks / Trade-offs

- **风险**: 测试依赖本地ES服务可用性
- **缓解**: 添加服务检查机制，如果ES不可用则跳过测试
- **权衡**: 使用真实ES服务而非mock，确保测试真实性但增加环境依赖
- **风险**: 网络连接问题可能导致测试失败
- **缓解**: 添加连接超时和重试逻辑
